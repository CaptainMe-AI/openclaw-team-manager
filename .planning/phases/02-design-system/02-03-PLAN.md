   - `error`: `string | undefined` (error message, replaces helperText, turns border red)

   ```
   <div className={cn("w-full", className)}>
     {label && <label>...</label>}
     <div className="relative">
       {icon && <span className="absolute left-3 top-1/2 -translate-y-1/2 text-text-secondary">...</span>}
       <input ... />
     </div>
     {error ? <p className="error">...</p> : helperText ? <p className="helper">...</p> : null}
   </div>
   ```

   Input element classes:
   - Base: `w-full bg-background border border-border rounded-md py-2 px-3 text-sm text-text-primary placeholder-text-secondary`

   - Error state: when `error` prop is truthy, replace border/focus classes with `border-danger focus:border-danger focus:ring-danger`
   - With icon: add `pl-9` to input (left padding for icon space)

   Label: `block text-sm font-semibold text-text-primary mb-1`
   Helper text: `mt-1 text-xs text-text-secondary`
   Error text: `mt-1 text-xs text-danger`

   Use `React.forwardRef` to wrap the component so parent components can focus the input programmatically.

2. Create `source/dashboard/app/frontend/components/ui/Table.tsx` (per D-01, D-03, D-11, D-13):

   Import cn from `@/lib/utils`.

   Export the ColumnDef type for consumers:
   ```typescript
   export interface ColumnDef<T> {
     header: string;
     accessor: keyof T | ((row: T) => React.ReactNode);
     align?: "left" | "center" | "right";
     className?: string;
   }
   ```

   Props interface (per UI-SPEC Table section):
   - `columns`: `ColumnDef<T>[]` (column definitions)
   - `data`: `T[]` (row data array)
   - `onRowClick`: `((row: T, index: number) => void) | undefined` (row click handler)
   - `className`: `string | undefined`

   The component must be generic: `export function Table<T>({ columns, data, onRowClick, className }: TableProps<T>)`

   Visual contract:
   - `<table>`: `w-full text-left border-collapse`
   - `<thead>` row: no explicit bg class on thead itself
   - `<tr>` in thead: `bg-surface-hover/50`
   - `<th>`: `p-4 text-xs text-text-secondary uppercase tracking-wider font-normal` with alignment via `text-left`, `text-center`, or `text-right`
   - `<tbody>`: `text-sm divide-y divide-border`
   - `<tr>` in tbody: `hover:bg-surface-hover/30 transition-colors` plus `cursor-pointer` when onRowClick is defined
   - `<td>`: `p-4` with alignment matching column def

   Cell rendering: If `accessor` is a function, call it with the row. If it's a key, render `row[accessor]` (cast to ReactNode). This keeps the Table generic.

   Wrap entire table in a `<div className="overflow-x-auto">` for horizontal scroll on mobile (per D-11 responsive consideration -- full responsive card layout is handled per-screen in later phases, but Table provides basic horizontal scroll as a baseline).
