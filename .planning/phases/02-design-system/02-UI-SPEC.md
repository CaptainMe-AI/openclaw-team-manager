# Phase 2 -- UI Design Contract

> Visual and interaction contract for the design system phase. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none (custom components with Tailwind v4 CSS-first config) |
| Preset | not applicable |
| Component library | none (standalone typed React components per D-01) |
| Icon library | Font Awesome 6.x (`@fortawesome/react-fontawesome`, `fa-solid` weight) -- deferred to Phase 3 |
| Font | Inter (UI text) + JetBrains Mono (data/metrics) via Google Fonts |

**Rationale:** Project uses HTML mockups as the visual source of truth. Components are custom-built to match mockup fidelity (D-04, D-05). No shadcn, Radix, or headless library -- Tailwind utility classes with `cn()` utility provide the styling layer (D-02, D-03).

## Spacing Scale

Declared values (must be multiples of 4). Extracted from mockup class patterns across all 7 screens.

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Inline padding (`py-1`, `gap-1`), badge vertical padding, badge dot gap |
| sm | 8px | Compact element spacing (`px-2`, `py-2`, `gap-2`), badge horizontal padding, button md vertical padding |
| md | 16px | Default element spacing (`p-4`), table cell padding, section gaps |
| lg | 24px | Card padding (`p-6`), section padding, layout column gaps (`gap-6`) |
| xl | 32px | Layout gaps between major sections (`gap-8`) |
| 2xl | 48px | Page-level vertical spacing |
| 3xl | 64px | Page-level horizontal margins |

Exceptions:
- 12px (`px-3`) -- button horizontal padding at md size and input horizontal padding. Tailwind default spacing value, multiple of 4.

### Accent Reserved-For List

Accent (#00d4aa) is used ONLY for:
- Primary action buttons ("New Task", "Save Configuration", "Approve All", "Approve" inline)
- Active sidebar navigation indicator (left border + text color)
- Input focus ring (`focus:border-accent focus:ring-accent`)
- Primary button glow effect (`shadow-[0_0_10px_rgba(0,212,170,0.3)]`)
- Link-style interactive text (rare)

Accent is NOT used for: backgrounds, body text, borders in non-focused state, status indicators (use success/info/warning/danger instead).

## Responsive Breakpoints

Desktop-first approach (D-09). Components render at desktop density by default, with responsive overrides for smaller viewports.

| Breakpoint | Width | Behavior |
|------------|-------|----------|
| Default (desktop) | >1024px | Full data density, all columns visible, grid layouts |
| `lg` | 1024px | Minor adjustments, sidebar still visible |
| `md` | 768px | Tables start showing fewer columns |
| `sm` | 640px | Tables switch to card/list layout, cards stack vertically |

Tailwind prefixes: `sm:`, `md:`, `lg:`, `xl:` (D-10).

## Interaction States

All interactive components must implement these states:

| State | Visual Treatment |
|-------|-----------------|
| Default | Base styling per component spec |
| Hover | Background shift or border change (see each component) |
| Focus | `ring-2 ring-accent ring-offset-2 ring-offset-background` (buttons, inputs) |
| Active/Pressed | `active:scale-[0.98]` on buttons (subtle press feedback) |
| Disabled | `opacity-50 cursor-not-allowed pointer-events-none` |

Transition: `transition-colors` on all hover/focus state changes. Duration: Tailwind default (150ms).
