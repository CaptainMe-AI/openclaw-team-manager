## Interaction Contracts

### Sidebar Navigation (SHEL-01)

**Desktop (>= 1024px):**
- Sidebar is always visible, fixed width `w-64` (256px).
- Sidebar background: `bg-surface`. Right border: `border-r border-border`.
- Logo area at top: Font Awesome `faRobot` icon (16px, `text-accent`) + "OpenClaw" text (14px, `font-semibold`, `text-text-primary`). Padding: `px-4 py-4`.
- Nav items: Vertical list below logo. Each item is a `<NavLink>` from react-router.
- Nav item default state: `text-text-secondary`, `px-4 py-2`, `flex items-center gap-3`.
- Nav item hover state: `bg-surface-hover`, `text-text-primary`.
- Nav item active state: `bg-surface-hover`, `text-accent`, `border-l-2 border-accent`. Per D-03.
- Dashboard NavLink uses `end` prop to prevent partial path matching. Per RESEARCH.md Pitfall 4.
- Approvals nav item shows a `Badge` with hardcoded count "3" to the right of the label. Per D-04.

**Mobile (< 1024px):**
- Sidebar is hidden by default. Hamburger button in TopBar toggles visibility via Zustand `useUIStore`.
- Sidebar slides in from the left as a drawer overlay. Animation: `transform translateX(-100%) -> translateX(0)`, duration `200ms`, easing `ease-out`.
- Semi-transparent backdrop overlay behind sidebar: `bg-black/50`. Clicking backdrop closes sidebar.
- Sidebar closes on navigation (watch `location.pathname` changes). Per RESEARCH.md Pitfall 5.
- Close button (`faXmark`) in top-right of sidebar drawer.

Source: D-01, D-02, D-03, D-04, SHEL-01, RESEARCH.md Patterns 2-3, Pitfalls 4-5.

### Top Bar (SHEL-02)

- Full width of remaining space (to the right of sidebar on desktop, full width on mobile).
- Background: `bg-surface`. Bottom border: `border-b border-border`.
- Height: `h-16` (64px).
- Padding: `px-4` on mobile, `px-6` on desktop.
- Layout: `flex items-center justify-between`.

**Left section:**
- Mobile only: Hamburger button (`faBars`, `w-5 h-5`). Visible below `lg` breakpoint.
- Breadcrumb component: "Command Center" (`text-text-secondary`) + `faChevronRight` (10px, `text-text-secondary`) + "{Page Name}" (`text-text-primary`, `font-semibold`). Per D-05.

**Center/Right section:**
- Search input: `w-64` on desktop, hidden on mobile (< 640px). Placeholder: `"Search agents, tasks, or approvals (Press '/')"`. Left icon: `faMagnifyingGlass`. Per D-06.
- Pressing `/` key focuses the input (skip if focus is already in INPUT, TEXTAREA, or SELECT). Per RESEARCH.md Pattern 6.
- Notification bell: `faBell` icon (`w-5 h-5`, `text-text-secondary`). Absolute-positioned dot indicator: `w-2 h-2 rounded-full bg-accent` at top-right of bell icon. Static, no click behavior. Per D-07.
- User avatar: Circular container `w-8 h-8 rounded-full bg-surface-hover`. Shows `faUser` icon (`text-text-secondary`) as placeholder. Click triggers UserMenu dropdown. Per D-08.

Source: D-05, D-06, D-07, SHEL-02, SHEL-03, RESEARCH.md Patterns 5-6.

### User Menu Dropdown (SHEL-04)

- Triggered by clicking the avatar button. Per D-09.
- Rendered inline (not portal) -- avatar is in top-right corner with no overflow clipping issues. Per RESEARCH.md Anti-Patterns.
- Positioned: `absolute right-0 top-full mt-2`. Width: `w-56` (224px).
- Background: `bg-surface`. Border: `border border-border`. Border radius: `rounded-lg`. Shadow: `shadow-lg`.
- Z-index: `z-50`.

**Dropdown content:**
1. Header section: Display name (`text-sm font-semibold text-text-primary`) + email (`text-xs text-text-secondary`). Padding: `px-4 py-3`. Bottom border: `border-b border-border`.
2. Menu items section: Padding: `py-1`.
   - "Profile" -- navigates to `/settings`. Icon: `faUser`. Text: `text-sm text-text-secondary`. Hover: `bg-surface-hover text-text-primary`.
   - "Settings" -- navigates to `/settings`. Icon: `faGear`. Text: `text-sm text-text-secondary`. Hover: `bg-surface-hover text-text-primary`.
3. Divider: `border-t border-border`.
4. "Sign Out" -- destroys Devise session via `DELETE /users/sign_out` with CSRF token, then redirects to `/users/sign_in` via `window.location.href`. Icon: `faRightFromBracket`. Text: `text-sm text-danger`. Hover: `bg-danger/10`.

**Close behavior:**
- Click outside dropdown: closes. Per D-09.
- Press Escape: closes. Per D-09.
- Implementation: `useClickOutside` hook pattern from RESEARCH.md Pattern 7.

Source: D-08, D-09, SHEL-04, RESEARCH.md Patterns 5 and 7.

### Routing (SHEL-05)

- `createBrowserRouter` with root layout route. Per D-12.
- Root route element: `<AppShell />` rendering `<Outlet />` for child routes. Per D-13.
- 6 child routes: `/` (index), `/agents`, `/tasks`, `/usage`, `/approvals`, `/settings`. Per D-12.
- Rails catch-all route added to `config/routes.rb` (AFTER Devise routes) for direct URL navigation. Per RESEARCH.md Pitfall 1.
- App.tsx changes from design system demo to `<RouterProvider router={router} />`. Per RESEARCH.md integration points.
- No page reload on navigation (SPA). All routes render within the app shell layout.

Source: D-12, D-13, SHEL-05, RESEARCH.md Patterns 1-2, Pitfall 1.

### Placeholder Skeleton (D-10, D-11)

- Single `PlaceholderSkeleton` component used on all 6 pages.
- Shimmer effect: `animate-pulse` on skeleton blocks.
- Layout: 4 KPI card skeletons in a responsive grid (`grid-cols-1 md:grid-cols-2 lg:grid-cols-4 gap-4`) + 1 table skeleton block below.
- KPI card skeleton: `bg-surface rounded-lg border border-border p-6`. Contains: title bar (12px height, 80px width, `bg-border rounded`) + value bar (24px height, 64px width, `bg-border rounded`).
- Table skeleton: `bg-surface rounded-lg border border-border p-6`. Contains: header bar (12px height, 128px width) + 5 rows of decreasing width bars.
- Each placeholder page renders: page title (Heading role) + page subtitle (Body role, `text-text-secondary`) + `<PlaceholderSkeleton />` with `mt-6` margin.

Source: D-10, D-11, RESEARCH.md skeleton code example.

## Responsive Behavior

| Breakpoint | Sidebar | TopBar | Main Content |
|------------|---------|--------|-------------|
| Mobile (< 640px) | Hidden, drawer overlay via hamburger | Full width, hamburger visible, search hidden | `p-4`, single column grid |
| Tablet (640-1024px) | Hidden, drawer overlay via hamburger | Full width, hamburger visible, search visible (`w-48`) | `p-4`, 2-column grid for skeletons |
| Desktop (>= 1024px) | Fixed visible `w-64` | Fills remaining width, hamburger hidden | `p-6`, 4-column grid for KPI skeletons |

Breakpoint implementation: Tailwind responsive prefixes (`sm:`, `md:`, `lg:`). The sidebar visibility toggle at `lg:` breakpoint (1024px).

Source: D-01, TEAM_MANAGER_SPEC.md Responsive Breakpoints, DSGN-04.

## State Management

| State | Store | Scope |
|-------|-------|-------|
| Sidebar open/closed (mobile) | Zustand `useUIStore` | Global -- hamburger button (TopBar) and sidebar (Sidebar) are sibling components |
| Dropdown open/closed | Local `useState` in UserMenu | Component-local -- only UserMenu needs this |
| Current route | React Router (URL) | URL-driven -- breadcrumb reads from `useLocation()`, NavLink reads automatically |

Source: RESEARCH.md Patterns 3 and 7, RESEARCH.md Anti-Patterns ("do not store route-derived state in Zustand").

## Accessibility Notes

- All NavLink items have visible text labels (not icon-only on desktop).
- Search input has an associated `aria-label` or visible label for screen readers.
- User menu dropdown button has `aria-expanded` and `aria-haspopup="true"`.
- Dropdown menu items are focusable and navigable via keyboard (Tab/Shift+Tab).
- Escape key closes the dropdown and returns focus to the trigger button.
- Mobile sidebar backdrop is an `aria-hidden` overlay; the sidebar itself is keyboard-navigable.
- All Font Awesome icons in nav items are decorative (`aria-hidden="true"`) since text labels are present.
