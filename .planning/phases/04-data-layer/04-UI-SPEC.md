# Phase 4 — UI Design Contract

> Visual and interaction contract for the data layer phase. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

**Phase classification:** Infrastructure / data pipeline. Phase 4 produces zero new visible components or screens. Its frontend deliverables are: React Query provider setup, custom data hooks, Zustand stores, TypeScript API types, and a shared fetch wrapper. These are consumed by downstream phases 5-10 for rendering actual UI.

This contract defines the **data-state visual patterns** (loading, error, empty) and **copywriting defaults** that hooks must support and downstream phases must render consistently.

## Copywriting Contract

Phase 4 establishes the **default copy patterns** that all downstream hooks and pages must use. No new screens render in this phase, but these values are embedded in TypeScript types, hook return patterns, and Zustand store defaults.

| Element | Copy |
|---------|------|
| Primary CTA | not applicable (no new UI actions in this phase) |
| Loading state | Render `PlaceholderSkeleton` component (pulse animation, no text) |
| Empty state heading | "No {resource} found" |
| Empty state body | "There are no {resource} to display yet. Data will appear here once {resource} are created." |
| Error state heading | "Failed to load {resource}" |
| Error state body | "Something went wrong while fetching {resource}. Check that the server is running and try again." |
| Error retry CTA | "Retry" |
| Pagination empty | "No more results" |
| Toast: API error | "{resource} could not be loaded. Please try again." |

**Resource name substitutions** (lowercase, plural):

| Hook | Resource Name |
|------|---------------|
| useAgents | "agents" |
| useTasks | "tasks" |
| useApprovals | "approvals" |
| useUsage | "usage records" |
| useDashboard | "dashboard data" |
| useSettings | "settings" |

### Destructive Actions

This phase exposes no destructive actions directly. The approve/deny API endpoints (PATCH) are non-destructive mutations. Downstream phases (7: Approvals) will define confirmation UX for those actions.

## Data State Patterns

Phase 4 hooks must return consistent state shapes that downstream phases render. These are the **required states per hook**:

| State | Condition | UI Pattern |
|-------|-----------|------------|
| Loading (initial) | `isLoading === true` | PlaceholderSkeleton component (existing) |
| Success (with data) | `data !== undefined && data.length > 0` | Render resource list/grid/chart |
| Success (empty) | `data !== undefined && data.length === 0` | Empty state card with heading + body from copywriting contract |
| Error | `isError === true` | Error card with heading + body + "Retry" button calling `refetch()` |
| Refetching | `isFetching && !isLoading` | Subtle indicator (no skeleton replacement -- show stale data with fetch indicator) |

### React Query Defaults

| Setting | Value | Rationale |
|---------|-------|-----------|
| staleTime | 30,000ms (30s) | Dashboard data refreshes frequently but not real-time |
| gcTime | 300,000ms (5min) | Keep cached data available for back-navigation |
| retry | 1 | One retry on failure, then show error state |
| refetchOnWindowFocus | true | Refresh data when operator returns to tab |

**Source:** 04-RESEARCH.md Pattern 4 and QueryClient setup example.

### Zustand Store Structure

| Store | State Keys | Purpose |
|-------|------------|---------|
| uiStore (existing) | sidebarCollapsed | Sidebar toggle state |
| filterStore (new) | agentFilters, taskFilters, approvalFilters, usageTimeRange | Page-level filter selections persisted across navigation |
| viewStore (new) | agentView ("grid" or "table"), taskView ("board" or "list") | View toggle state per resource page |

**Source:** 04-RESEARCH.md Pattern 6, CONTEXT.md D-32.

## API Response Rendering Contract

All API responses follow a consistent shape that downstream UI components consume:

### Paginated List Response

```
{
  data: T[],
  pagination: {
    current_page: number,
    per_page: number,
    total_pages: number,
    total_count: number
  }
}
```

### Pagination UI Values

| Element | Value |
|---------|-------|
| Default per_page | 25 |
| Max per_page | 100 |
| Page param name | "page" |
| Per-page param name | "per_page" |

### Sort UI Values

| Element | Value |
|---------|-------|
| Sort param name | "sort" |
| Direction param name | "dir" |
| Default direction | "desc" |
| Sort indicator (ascending) | Up arrow icon |
| Sort indicator (descending) | Down arrow icon |

**Source:** CONTEXT.md D-25, D-26, 04-RESEARCH.md Pagy configuration.
