## Loading & Empty & Error States

### Loading State

When `useAgents` is in `isLoading` state:
- Grid view: Show 8 skeleton cards using `PlaceholderSkeleton` in the grid layout
- Table view: Show skeleton rows (8 rows with pulsing `bg-surface-hover/50` backgrounds)
- Filters and view toggle render normally (interactive immediately)

**Skeleton card visual:**
- Same dimensions as AgentCard
- Pulsing `animate-pulse` background blocks where text/sparkline would be
- Uses `bg-surface-hover/50 rounded` blocks for content placeholders

### Empty State (No Agents)

When `data.data` is an empty array (no agents match filters):

| Element | Copy |
|---------|------|
| Icon | `faRobot` at 48px, `text-text-secondary opacity-50` |
| Heading | "No agents found" |
| Body | "No agents match the current filters. Try adjusting your filter criteria or check that agents are registered." |
| Action | `Button variant="secondary"` -- "Clear Filters" (calls `resetAgentFilters()`) |

**Visual:**
- Centered in content area: `flex flex-col items-center justify-center py-16 text-center`
- Icon: `text-5xl text-text-secondary opacity-50 mb-4`
- Heading: `text-lg font-semibold text-text-primary mb-2`
- Body: `text-sm text-text-secondary max-w-md mb-6`
- Action: centered Button below body text

### Error State

When `useAgents` returns `isError`:

| Element | Copy |
|---------|------|
| Icon | `faExclamationTriangle` at 48px, `text-danger opacity-50` |
| Heading | "Failed to load agents" |
| Body | "Unable to fetch agent data from the server. Check that the Rails server is running and try again." |
| Action | `Button variant="primary"` -- "Retry" (calls `refetch()`) |

**Visual:** Same centered layout as empty state, but icon uses `text-danger`.

### Destructive Actions

| Action | Trigger | Confirmation Approach |
|--------|---------|----------------------|
| Disable Agent | Context menu "Disable Agent" click | No confirmation dialog -- action fires immediately with toast notification. Agent status updates to "disabled" via PATCH. Reversible by restarting (not destructive-permanent). |

**Rationale:** Disable is reversible (restart undoes it). In a local tool context, a confirmation dialog adds friction without safety benefit. The toast provides undo-awareness.
