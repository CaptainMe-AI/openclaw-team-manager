## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline badge padding, gap between sortable cards |
| sm | 8px | Compact element spacing, column inner padding, card internal gaps |
| md | 16px | Default element spacing, card padding, column header margin, modal action row gap |
| lg | 24px | Section padding, page header to filters gap, modal action row top margin |
| xl | 32px | Not used this phase |
| 2xl | 48px | Not used this phase |
| 3xl | 64px | Not used this phase |

Exceptions:
- Kanban column width: 280px (not on the 4px grid -- functional minimum for card readability with 2-line descriptions)
- Kanban column min-height: 200px (ensures empty columns remain visible drop targets)
- Drag activation distance: 8px (PointerSensor constraint to distinguish click from drag)
- Touch target minimum: 44px (context menu triggers, modal close button)

Source: Design screenshot (screenshot_5.png), established pattern from Phase 5 AgentCard.

## Interaction Contracts

### Kanban Board

The Kanban board is the primary visual anchor of the Tasks page. It occupies the full content area below the filter bar and is the default view on page load.

| Interaction | Visual Feedback | Implementation |
|-------------|-----------------|----------------|
| Hover task card | Border transitions to text-secondary color | `hover:border-text-secondary transition-colors` |
| Start dragging card (8px movement threshold) | Original card reduces to 50% opacity; DragOverlay shows card clone with shadow-lg and 2deg rotation | `isDragging ? opacity-50 : 1`, DragOverlay with `shadow-lg rotate-[2deg]` |
| Drag over valid column | Column shows accent ring at 30% opacity | `isOver && "ring-2 ring-accent/30"` |
| Drop card in new column | Card appears in target column; optimistic PATCH to API; toast on failure + rollback | useMutation with onMutate/onError rollback |
| Drop card in same column | No status change; card returns to original position | onDragEnd checks if container changed |
| Keyboard drag (Tab to card, Space to pick up, arrows to move) | Same visual feedback as pointer drag | KeyboardSensor with sortableKeyboardCoordinates |
| Horizontal scroll on overflow | Native scrollbar (styled via existing scrollbar CSS) | `overflow-x-auto` on column container |

### View Toggle

The TaskViewToggle is icon-only. It renders two buttons: `faGrip` (board) and `faList` (list). No text labels are shown. Accessibility is provided via `aria-label` and `aria-pressed`.

| Interaction | Visual Feedback |
|-------------|-----------------|
| Click board icon (faGrip) | Board icon button gets accent background; list view hides, board view appears |
| Click list icon (faList) | List icon button gets accent background; board view hides, list view appears |

### Filters

| Interaction | Visual Feedback |
|-------------|-----------------|
| Click filter option | Active option gets accent background (variant="primary"); previously active reverts to secondary |
| Click "All" in any filter group | Clears that filter dimension; "All" button gets accent background |

### New Task Modal

| Interaction | Visual Feedback |
|-------------|-----------------|
| Click "New Task" button | Modal slides in with backdrop blur (bg-black/60 backdrop-blur-sm) |
| Click backdrop or "Discard Task" | Modal closes, form state discarded |
| Click X button | Modal closes, form state discarded |
| Submit with missing required field | Red border on invalid inputs; error message below each invalid field |
| Submit valid form | Loading state on "Create Task" button (disabled + spinner); success toast + close; error toast on failure |
| Click attachment area | Toast: "File attachments coming soon" (placeholder for v1) |
| Priority button group | Active priority gets accent/highlighted border (per screenshot: dashed amber border on selected) |

### Priority Button Group (in modal)

Per design screenshot, the priority selector uses a 4-button group: "-" (none/unset), Low, Medium (default), High.

| Button | Maps to | Visual when selected |
|--------|---------|---------------------|
| - | Not used (P3 minimum) | Outlined dashed border |
| Low | priority 3 (P3) | Outlined dashed border |
| Medium | priority 2 (P2, default) | Accent dashed border (highlighted) |
| High | priority 1 (P1) | Outlined dashed border |

Note: P0 (Critical) is not available in the creation form -- it is set via direct status update only. The form uses the labels from the design spec: "-", "Low", "Medium", "High".

## Task Card Visual Contract

Each task card in the Kanban board follows this exact structure:

```
+--[4px priority-colored left border]---------------------------+
| [task_id mono 12px secondary]          [Priority Badge P0-P3] |
| [title 14px semibold primary, single line truncate]           |
| [description 12px secondary, 2-line clamp]                    |
|                                                                |
| [robot icon] [agent_name 12px secondary]    [timestamp mono]  |
+----------------------------------------------------------------+
```

CSS classes for the card container:
- `bg-surface rounded-lg border border-border border-l-4 border-l-priority-{N} p-3`
- Hover: `hover:border-text-secondary transition-colors` (only the non-left borders transition)
- When dragging (isDragOverlay): `shadow-lg rotate-[2deg]`
- When being dragged (isDragging): `opacity-50`
- Card width: fills column width (280px minus column padding)
- Card gap: 8px between cards in a column

Agent display:
- If agent assigned: FontAwesome `faRobot` icon + agent_name
- If unassigned: FontAwesome `faUserXmark` icon + "Unassigned" in italics

Timestamp: relative time from `created_at` using `formatDistanceToNow` from date-fns, displayed in JetBrains Mono.

Field spacing: 16px (mb-4) between each form field group.
Action row: `flex justify-end gap-4 mt-6 pt-4 border-t border-border`.

Textarea: 4 rows minimum, same styling as Input but using `<textarea>`.

Attachment area:
- Dashed border (`border-2 border-dashed border-border`), rounded-lg

- Centered text: "Drag files, context, requirements, or special instructions"
- Text: 12px, text-secondary
- On click: show toast "File attachments coming soon"

## Responsive Behavior

| Breakpoint | Layout |
|------------|--------|
| Desktop (>1024px) | All 6 columns visible, may need horizontal scroll at ~1320px if sidebar expanded |
| Tablet (640-1024px) | Horizontal scroll required; columns maintain 280px min-width |
| Mobile (<640px) | Horizontal scroll; columns stack option not used -- horizontal swipe/scroll is the standard Kanban pattern |

The column container uses: `flex gap-2 overflow-x-auto pb-2` (pb-2 ensures scrollbar doesn't clip content).

Filter bar wraps on narrow viewports using `flex-wrap`.

Modal is responsive via `max-w-lg mx-4` -- on mobile it fills available width minus 16px margins.

## Accessibility Contract

| Element | Requirement |
|---------|-------------|
| Kanban drag-and-drop | KeyboardSensor enabled; Space to pick up, arrows to move, Space to drop |
| DragOverlay | `aria-grabbed` attribute on active card |
| Modal | `role="dialog"`, `aria-modal="true"`, `aria-labelledby` pointing to title |
| Modal focus trap | First focusable element (Task Name input) receives focus on open; Tab cycles within modal |
| Modal close | Escape key closes modal |
| View toggle buttons | Icon-only; each button has `aria-label` ("Board view" / "List view") and `aria-pressed` reflecting active state |
| Filter buttons | `aria-pressed` attribute reflecting active state |
| Form validation | `aria-invalid` on invalid inputs; `aria-describedby` linking to error messages |
| Priority button group | `role="radiogroup"` with `role="radio"` and `aria-checked` per button |
| Column task count | Visually displayed; screen readers get column label + count via aria-label on column |
