Note on history: Fetch all approvals without status filter, then client-side filter to exclude pending. This is the simplest approach for v1 mock data (per research recommendation option 2). The API only supports single-value status filter, so fetching all and filtering client-side avoids needing two separate queries merged together.

Batch approve:
```typescript
const batchApprove = useBatchApprove();

function handleBatchApprove() {
  const ids = pendingApprovals.map((a) => a.id);
  batchApprove.mutate(ids, {
    onSuccess: (data) => {
      toast.success(`${data.approved} approvals granted`);
      setExpandedId(null);
    },
    onError: () => {
      toast.error("Failed to process batch approval");
    },
  });
}
```
