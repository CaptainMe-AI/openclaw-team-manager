## Spacing Scale

| Token | Value | Usage |

| xs | 4px | Icon gaps, inline badge padding, gap between approve/deny buttons |
| sm | 8px | Compact element spacing, card internal row gaps, filter button gaps |

| 2xl | 48px | Not used this phase |
| 3xl | 64px | Not used this phase |

Exceptions:
- Touch target minimum: 44px (approve/deny buttons, tab buttons)
- Detail panel split: 60%/40% left-right ratio (reasoning panel / context panel)

## Typography

| Role | Size | Weight | Line Height | Font | Usage in this phase |
|------|------|--------|-------------|------|---------------------|
| Body | 14px (text-sm) | 400 (normal) | 1.5 | Inter | Approval descriptions, filter labels, detail panel reasoning text, history table cells |
| Label | 12px (text-xs) | 400 (normal) | 1.5 | Inter | Time waiting display, agent name metadata, context field labels, tab inactive state |
| Heading | 20px (text-xl) | 600 (semibold) | 1.2 | Inter | Page title "Approvals" |
| Data | 12px (text-xs) | 400 (normal) | 1.5 | JetBrains Mono | Timestamps, commands in context panel, file paths, budget amounts |

Additional typography rules:
- Card title: 14px (text-sm) at weight 600 (font-semibold), Inter
- Page subtitle: 14px (text-sm) at weight 400 (normal), text-secondary, Inter
- Tab labels: 14px (text-sm) at weight 600 (font-semibold) when active, weight 400 when inactive, Inter
- Risk badge text: 10px (text-[10px]) at weight 400, Inter (existing Badge component convention)
- Approval type label: 12px (text-xs) at weight 400, uppercase, tracking-wider, text-secondary, Inter
- Detail panel section headings: 12px (text-xs) at weight 600 (font-semibold), uppercase, tracking-wider, Inter
- History table headers: 12px (text-xs) at weight 400, uppercase, tracking-wider, text-secondary, Inter (existing Table convention)
- "Approve All" button: 14px (text-sm) at weight 600, Inter (matches Button md size)

Declared scale: 10, 12, 14, 20 (4 sizes). The 10px size is an exception inherited from the existing Badge component convention and is only used in risk badge text.

Source: Existing component patterns from Phases 5-6 (AgentCard, TaskCard, AgentFilters, TaskListView).

## Interaction Contracts

### Tab Navigation (Pending / History)

Two tabs at the top of the approval content area. Uses local component state, not routing.

| Interaction | Visual Feedback | Implementation |
|-------------|-----------------|----------------|
| Click "Pending" tab | Tab gets accent bottom border (2px border-b-accent), text becomes text-primary weight 600; other tab reverts to text-secondary weight 400 | `activeTab` state: `'pending' \| 'history'` |
| Click "History" tab | Tab gets accent bottom border, text-primary, weight 600; Pending tab reverts | Same state toggle |
| Default state on page load | "Pending" tab is active | `useState<'pending' \| 'history'>('pending')` |

Tab styling: Each tab is a button with `px-4 py-2 text-sm` base. Active: `border-b-2 border-accent text-text-primary font-semibold`. Inactive: `border-b-2 border-transparent text-text-secondary font-normal hover:text-text-primary`. Tab row has `border-b border-border` full width.

### Pending Tab -- Approval Cards

| Interaction | Visual Feedback | Implementation |
|-------------|-----------------|----------------|
| Hover approval card | Border transitions to text-secondary | `hover:border-text-secondary transition-colors` (inherited from Card) |
| Click card body (not buttons) | Card expands to show detail panel below; chevron rotates from right to down | `expandedId` state; only one card expanded at a time |
| Click already-expanded card body | Card collapses; chevron rotates from down to right | `setExpandedId(null)` |
| Click "Approve" button | `e.stopPropagation()` to prevent expand toggle; optimistic removal from list; success toast "Approval granted"; on API error: rollback + error toast | `useApproveApproval` mutation |
| Click "Deny" button | `e.stopPropagation()`; optimistic removal from list; success toast "Approval denied"; on API error: rollback + error toast | `useDenyApproval` mutation |
| Click "Approve All" button | All visible pending approvals approved; success toast "N approvals granted"; disabled during mutation (loading spinner) | `useBatchApprove` mutation with collected IDs |
| "Approve All" with 0 pending items | Button is disabled (opacity-50, cursor-not-allowed) | Conditional `disabled` prop |

### Approval Card Collapsed State

```
+-- Card (bg-surface rounded-lg border border-border p-4) ---+
| [chevron >] [risk badge] [title 14px semibold]              |
|            [type label 12xs uppercase] [time waiting mono]   |
|                                                              |
| [agent icon] [agent name 12xs]    [Approve btn] [Deny btn]  |
+--------------------------------------------------------------+
```

- Chevron: FontAwesome `faChevronRight` (collapsed) / `faChevronDown` (expanded), 12px, text-secondary, transitions with `transition-transform duration-200`
- Risk badge: Badge variant="status" with risk-to-color mapping
- Title: 14px, font-semibold, text-primary, truncate on single line
- Type label: 12px, uppercase, tracking-wider, text-secondary (e.g., "PRODUCTION DEPLOYMENT")
- Time waiting: 12px, font-mono, text-secondary (e.g., "2 hours")
- Agent: FontAwesome `faRobot` icon + agent_name, 12px, text-secondary
- Approve button: Custom styled -- `bg-success/10 text-success border border-success/20 hover:bg-success/20 px-3 py-1 text-xs rounded-md font-semibold`
- Deny button: Custom styled -- `bg-danger/10 text-danger border border-danger/20 hover:bg-danger/20 px-3 py-1 text-xs rounded-md font-semibold`

### Filters (Pending tab only)

| Interaction | Visual Feedback |
|-------------|-----------------|
| Click risk level filter option | Active option gets accent background (variant="primary"); previously active reverts to secondary |
| Click "All" in risk filter | Clears risk filter; "All" button gets accent background |
| Sort by time waiting | Default sort is newest-first (longest waiting at top); toggle reverses order |

Risk level filter follows the button-group pattern established by AgentFilters and TaskFilters:
```
[All] [Low] [Medium] [High] [Critical]
```

Sort control: A single "Sort: Oldest First / Newest First" toggle button with FontAwesome `faSortAmountDown` / `faSortAmountUp` icon.

| Interaction | Visual Feedback |
|-------------|-----------------|
| Switch to History tab | Pending cards replaced by history table; filters hidden; "Approve All" hidden |

| No sorting in history table | Static display, no interactive sort headers (simplicity for v1) |

History table columns:

| Column | Width | Content |

| Decision | w-28 | Badge: "Approved" (success) or "Denied" (danger) |

## Responsive Behavior

| Breakpoint | Layout |
|------------|--------|
| Desktop (>1024px) | Full layout. Detail panel uses grid-cols-5 (3+2 split). All history table columns visible. |
| Tablet (640-1024px) | Detail panel stacks vertically (grid-cols-1). History table hides "Decided By" column. Filter bar wraps. |
| Mobile (<640px) | Detail panel stacks vertically. History table shows only Timestamp, Action, Decision columns. Approve/Deny buttons stack below card content. Tab labels remain horizontal. |

Card list is always full-width, single-column. No grid layout needed (unlike agent cards).

Filter bar wraps on narrow viewports using `flex-wrap`.

## Accessibility Contract

| Element | Requirement |
|---------|-------------|
| Tab navigation | `role="tablist"` on tab container; each tab has `role="tab"`, `aria-selected`, `aria-controls` pointing to tab panel |
| Tab panels | `role="tabpanel"`, `aria-labelledby` pointing to corresponding tab, `tabindex="0"` |
| Expandable cards | Card body has `role="button"`, `aria-expanded` (true/false), `aria-controls` pointing to detail panel ID |
| Card keyboard interaction | Enter/Space toggles expansion on focused card |
| Approve/Deny buttons | Standard `<button>` elements; `aria-label="Approve {title}"` and `aria-label="Deny {title}"` for screen reader context |
| Approve All button | `aria-label="Approve all {count} pending approvals"`; `aria-disabled="true"` when count is 0 |
| Filter buttons | `aria-pressed` attribute reflecting active state |
| Risk level filter group | `role="group"` with `aria-label="Filter by risk level"` |
| Detail panel | `id` matching card's `aria-controls`; content is read naturally in document order |
| History table | Standard `<table>` with `<thead>`, `<th scope="col">` headers |
| Toast notifications | sonner handles `role="status"` and `aria-live="polite"` automatically |
