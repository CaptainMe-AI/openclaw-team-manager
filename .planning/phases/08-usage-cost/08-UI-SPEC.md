### Semantic Colors for Trend Indicators

| Condition | Color | Token |
|-----------|-------|-------|
| Throughput increase (tokens, API calls up) | #10b981 | `--color-success` |
| Throughput decrease (tokens, API calls down) | #ef4444 | `--color-danger` |
| Cost increase (more expensive) | #ef4444 | `--color-danger` |
| Cost decrease (cheaper) | #10b981 | `--color-success` |
| Latency increase (slower) | #ef4444 | `--color-danger` |
| Latency decrease (faster) | #10b981 | `--color-success` |

Per D-06: green for improvement, red for degradation. "Improvement" means: more tokens/calls = good (green), less cost = good (green), less latency = good (green).

Accent reserved for: Export Report CTA button, active time period selector button. NOT used for chart elements (charts use the 5-color chart palette).

## KPI Card Specification

Each KPI card follows this internal layout:

```
[Icon circle]  [Title]                    [Trend badge]
               [Value] / [period label]
```

### KPI Card Details

| Card | Icon | Icon BG Color | Value Format | Period Label | Trend Logic |
|------|------|---------------|--------------|-------------|-------------|
| Total Tokens | faCoins | chart-1 (#00d4aa) at 10% opacity | Compact number (42.8M) | "/ 24h" (matches selected period) | Green up = more throughput |
| Total API Calls | faNetworkWired | chart-2 (#3b82f6) at 10% opacity | Compact number (124.5k) | "/ 24h" | Green up = more throughput |
| Estimated Cost | faDollarSign | chart-5 (#f59e0b) at 10% opacity | Currency ($842.50) | "/ 24h" | Red up = more expensive |
| Avg Latency | faStopwatch | chart-3 (#8b5cf6) at 10% opacity | Duration (1.8s or 245ms) | "avg" | Green down = faster |

### KPI Icon Circle Spec

- Size: 40px (w-10 h-10)
- Shape: rounded-lg
- Background: Chart color at 10% opacity (e.g., `bg-chart-1/10`)
- Icon size: 16px
- Icon color: Matching chart color (e.g., `text-chart-1`)

### KPI Trend Badge Spec

- Font: 12px Inter, weight 600
- Arrow: Unicode up-arrow or down-arrow character, or FontAwesome faArrowUp/faArrowDown
- Color: Green (#10b981) for improvement, Red (#ef4444) for degradation
- Format: "up-arrow 12.5%" or "down-arrow 1.2s"
- Null/zero trend: Show "-- " in text-secondary color

## Interaction Contract

| Interaction | Behavior |
|-------------|----------|
| Time period button click | Updates filterStore.usageTimeRange, triggers refetch of summary + charts data via TanStack Query |
| Chart tooltip hover | Shows dark-themed tooltip with data values at cursor position |
| Chart legend click | Recharts default: toggles series visibility |
| Export Report click | Generates CSV from current summary data, triggers browser download, shows success toast |
| Error retry click | Calls refetch() on TanStack Query hooks |
| Page load | Fetches summary + charts data for default time range (24h) |

### Data Fetching Strategy

| Hook | Endpoint | staleTime | Trigger |
|------|----------|-----------|---------|
| useUsageSummary | GET /api/v1/usage/summary?from=&to= | 30s | Time period change |
| useUsageCharts | GET /api/v1/usage/charts?from=&to=&granularity= | 30s | Time period change |

Both hooks derive `from` and `to` ISO timestamps from filterStore.usageTimeRange. Granularity auto-selected: "hour" for 1h/6h/24h, "day" for 7d/30d.
