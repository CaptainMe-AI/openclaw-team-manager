## Color

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | #0f1219 (`--color-background`) | Page background behind all content |
| Secondary (30%) | #1a1f2e (`--color-surface`) | KPI cards, recent tasks table rows, action required sidebar cards, timeline container |
| Accent (10%) | #00d4aa (`--color-accent`) | "New Task" button, active time period dropdown option, "View All" link hover state |
| Destructive | #ef4444 (`--color-danger`) | Deny button, failed event timeline dots, pending approval trend increase (red up-arrow), error state icon |

### Timeline Event Dot Colors

| Event Type | Color Token | Hex | StatusDot Status |
|------------|-------------|-----|------------------|
| Completed / Active / Approved | `--color-success` | #10b981 | success |
| Started / Task event | `--color-info` | #3b82f6 | info |
| Waiting (approval pending) | `--color-warning` | #f59e0b | warning |
| Failed / Error / Denied | `--color-danger` | #ef4444 | danger |
| Idle / Disabled | `--color-text-secondary` | #9ca3af | idle |

Source: Matches TEAM_MANAGER_SPEC.md Section 1.3 color mapping. Uses existing StatusDot component status values.

### Semantic Colors for Dashboard Trend Indicators

| KPI | Up Direction | Down Direction |
|-----|-------------|----------------|
| Active Agents | Green (more = good) | Red (fewer = bad) |
| Tasks in Progress | Green (more = good) | Red (fewer = bad) |
| Pending Approvals | Red (more = bad, inverted) | Green (fewer = good, inverted) |
| Tokens Used (24h) | Green (more throughput = good) | Red (less throughput = bad) |

Per Phase 8 pattern: trendInverted flag controls color semantics. Pending Approvals uses inverted logic (more pending = degradation = red).

Accent reserved for: "New Task" CTA button, active state in time period dropdown. NOT used for timeline dots or table elements.

Each KPI card follows the exact internal layout from UsageKpiCards.tsx:

```

               [Value]
```

### KPI Card Details

| Card | Icon | Icon BG Color | Value Format | Trend Logic |
|------|------|---------------|--------------|-------------|
| Active Agents | faRobot | success (#10b981) at 10% opacity | Plain integer (42) | Green up = more agents active |
| Tasks in Progress | faSpinner | info (#3b82f6) at 10% opacity | Plain integer (18) | Green up = more tasks running |
| Pending Approvals | faShieldHalved | warning (#f59e0b) at 10% opacity | Plain integer (3) | Red up = more pending (inverted) |
| Tokens Used (24h) | faCoins | chart-3 (#8b5cf6) at 10% opacity | Compact number (1.2M) | Green up = more throughput |

### KPI Icon Circle Spec

- Size: 40px (w-10 h-10)
- Shape: rounded-lg
- Background: Semantic color at 10% opacity
- Icon size: 16px (text-base)
- Icon color: Matching semantic color

### KPI Trend Badge Spec

Same as UsageKpiCards TrendBadge:
- Font: 12px Inter, weight 600
- Arrow icon: faArrowUp or faArrowDown
- Color: Green (#10b981) for improvement, Red (#ef4444) for degradation
- Format: "12.5%" with up/down arrow
- Null/zero trend: Show "--" in text-secondary color
- trendInverted: true for Pending Approvals (more = bad)

### KPI Skeleton Loading

4 skeleton cards matching KPI card dimensions (same pattern as UsageKpiCards SkeletonKpiCard):

## Activity Timeline Specification

### Layout

```
[Card]
  [Header: "Agent Activity Timeline" -- text-sm font-semibold text-text-primary]
  [Timeline container -- overflow-x-auto, py-4]
    [Time axis: labels at left (start) and right (now)]
    [Dot row: horizontally positioned StatusDot components]
```

### Timeline Properties

| Property | Value |
|----------|-------|
| Container | Card component with default variant, p-6 |
| Dot component | StatusDot size="md" (8px diameter) |
| Dot spacing | Positioned proportionally along timeline width based on occurred_at timestamp |
| Hover behavior | CSS tooltip (::after pseudo or positioned div) showing agent name, event label, relative timestamp |
| Time axis labels | Left: start of period (formatted with date-fns "HH:mm" or "MMM dd"), Right: "Now" |
| Max events displayed | 20 (limited server-side per DashboardService) |
| Scroll behavior | Horizontal scroll on overflow (overflow-x-auto) |
| Min width | 600px to prevent dots from overlapping on narrow containers |
| Height | 80px total (dot row + time labels + padding) |

### Timeline Dot Color Mapping

## Action Required Sidebar Specification

### Layout

```
[Section heading: "Action Required" -- text-sm font-semibold text-text-primary, mb-4]
[Approval cards stack -- space-y-3]
  [ApprovalCard (compact)] -- reuse existing component
  [ApprovalCard (compact)]
  [ApprovalCard (compact)]
  ...up to 5
```

### Compact ApprovalCard Usage

Reuse the existing ApprovalCard component directly:
- Pass `isExpanded={false}`
- Pass `onToggleExpand` as a no-op function (dashboard does not expand approval details; clicking navigates the user to /approvals for full detail)
- The card already handles: risk badge, title (truncated), approval type label, time waiting, agent name, approve/deny buttons with mutation hooks and toast feedback
- ApprovalCard already uses `min-w-0` and `truncate` on the title for overflow safety

### Sidebar Empty State

When no pending approvals:
- Icon: faShieldHalved, 32px, text-text-secondary opacity-50, centered
- Heading: "All clear" -- text-sm font-semibold text-text-primary
- Body: "No pending approvals require your attention." -- text-xs text-text-secondary
- Layout: flex flex-col items-center justify-center, py-8

### Approve/Deny Interaction in Sidebar

| Action | Behavior |
|--------|----------|
| Click "Approve" | useApproveApproval mutation fires, toast "Approval granted", invalidates ["dashboard"] and ["approvals"] query keys |
| Click "Deny" | useDenyApproval mutation fires, toast "Approval denied", invalidates ["dashboard"] and ["approvals"] query keys |
| Mutation pending | Button shows faSpinner spin, both buttons disabled |
| Success | Card removed from sidebar on next refetch (pending count decreases) |

Source: These behaviors are already implemented in ApprovalCard.tsx and useApprovals.ts hooks. No new mutation logic needed.

## Interaction Contract

| Interaction | Behavior |
|-------------|----------|
| Time period dropdown change | Updates time period state, triggers refetch of dashboard data via useDashboard(timePeriod) |
| "New Task" button click | Opens NewTaskModal overlay |
| Task row click | Toggles expandable row (shows description) |
| "View All" link click | Navigates to /tasks via React Router |
| Timeline dot hover | Shows tooltip with agent name, event label, relative timestamp |
| Approve button click | useApproveApproval mutation, toast, cache invalidation |
| Deny button click | useDenyApproval mutation, toast, cache invalidation |
| Retry button click | Calls refetch() on dashboard query |
| Page load | Fetches dashboard data for default time period (24h) |

### Data Fetching Strategy

| Hook | Endpoint | staleTime | Trigger |
|------|----------|-----------|---------|
| useDashboard | GET /api/v1/dashboard?time_period={period} | 15s | Time period change, page mount |
| useApproveApproval | PATCH /api/v1/approvals/:id/approve | N/A (mutation) | Approve button click |
| useDenyApproval | PATCH /api/v1/approvals/:id/deny | N/A (mutation) | Deny button click |

Dashboard uses 15s staleTime (per existing useDashboard hook) for more frequent KPI refresh compared to the 30s global default.

## Number Formatting Contract

| Value Type | Formatter | Example Input | Example Output |
|------------|-----------|---------------|----------------|
| Agent count | Plain integer | 42 | 42 |
| Task count | Plain integer | 18 | 18 |
| Approval count | Plain integer | 3 | 3 |
| Token count | formatCompact (Intl compact) | 1200000 | 1.2M |
| Cost (cents to dollars) | formatCost | 4250 | $42.50 |
| Duration | date-fns differenceInMinutes/hours or custom | 7200000ms | "2h 15m" |
| Duration unknown | Em-dash | null | "--" |
| Relative timestamp | date-fns formatDistanceToNow | Date | "2h ago" |
| Trend percentage | formatTrend | 12.5 | "12.5%" |

Font for all data values: JetBrains Mono (font-mono). Font for labels/units: Inter (font-sans).

Source: Existing formatters from lib/formatters.ts. Count KPIs (agents, tasks, approvals) display as plain integers since they are small numbers (not millions).
